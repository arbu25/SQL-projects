## О репозитории

Этот репозиторий содержит выполненные мной практические задания по **SQL** и **проектированию реляционных баз данных**. Материалы охватывают путь от создания таблиц и простых `SELECT`-запросов до рекурсивных CTE, оконных функций, регулярных выражений, нормализации и разработки схем баз данных на основе ER-диаграмм.

Каждый SQL-файл посвящён отдельному набору тем и содержит решения соответствующих задач.

## Основные технологии и навыки

| Направление | Используемые инструменты и конструкции |
|---|---|
| СУБД | PostgreSQL |
| DDL | `CREATE TABLE`, `CREATE SEQUENCE`, ограничения целостности |
| DML | `INSERT`, `UPDATE`, `DELETE` |
| Выборка данных | `SELECT`, `WHERE`, `BETWEEN`, `IN`, `LIKE`, `ILIKE` |
| Объединение таблиц | `INNER JOIN`, `LEFT JOIN`, `SELF JOIN` |
| Агрегация | `GROUP BY`, `HAVING`, агрегатные функции |
| Подзапросы | скалярные, многострочные и коррелированные подзапросы |
| Продвинутый SQL | CTE, `WITH RECURSIVE`, оконные функции, `LATERAL` |
| Работа с текстом | строковые функции и POSIX-регулярные выражения |
| Проектирование БД | ER-моделирование, нормализация от 1НФ до НФБК |
| Надёжность данных | первичные и внешние ключи, `UNIQUE`, `CHECK`, `NOT NULL`, транзакции |

## 1. CREATE TABLE — Data Types & Integrity Constraints

**Темы:** типы данных, первичные и внешние ключи, уникальность, значения по умолчанию и проверки `CHECK`.

<details>
<summary><strong>Условие задачи</strong></summary>

### Таблица «Страны»

Необходимо создать таблицу со следующими полями:

- идентификатор страны — непустое целое число и первичный ключ;
- название страны — непустая строка длиной не более 64 символов.

### Таблица «Товары»

Необходимо создать таблицу со следующими полями:

- идентификатор товара — непустое целое число и первичный ключ;
- название — уникальная непустая строка длиной не более 2000 символов;
- ряд на складе — положительное целое число, содержащее не более трёх цифр;
- место в ряду — положительное целое число, содержащее не более трёх цифр;
- поставщик — непустая строка длиной не более 2000 символов;
- значение поставщика по умолчанию — `ОАО Царицынский мясокомбинат`;
- количество — непустое положительное целое число длиной не более трёх цифр;
- цена — непустое число с точностью до двух знаков после запятой и не более чем девятью цифрами до запятой;
- идентификатор страны — внешний ключ на таблицу «Страны».

</details>

## 7. Database Design Project — Music Library

**Темы:** ER-моделирование, создание схемы БД, объединения таблиц и агрегация.

### Предметная область

Проект представляет базу данных музыкальной библиотеки, которая включает:

- исполнителей;
- музыкальные композиции;
- альбомы;
- жанры;
- связи между композициями и исполнителями.

Исполнитель описывается именем или названием, типом исполнителя, описанием и основным жанром. Композиция содержит название, продолжительность, жанр и связь с исполнителем. Альбом содержит название и основной жанр. Каждая композиция входит только в один альбом.

### Структура папки

`1.Database Design Music: ER-Modeling, DDL Schema Implementation`

| Файл | Содержание |
|---|---|
| `ER-Diagram` | ER-диаграмма предметной области музыкальной библиотеки |
| `1.SQL script created according to the ER diagram` | DDL-скрипт создания таблиц, связей и ограничений на основе ER-диаграммы |
| `Multi-table Joins & Aggregation — Album Catalog Analysis...` | SQL-запросы для анализа альбомов, композиций и исполнителей |

<details>
<summary><strong>Реализованные аналитические запросы</strong></summary>

1. Вывод списка всех альбомов, входящих в них композиций и исполнителей.
2. Подсчёт для каждого альбома числа композиций с несколькими исполнителями.
3. Поиск альбомов с максимальным количеством композиций продолжительностью более трёх минут.
4. Перенос всех композиций из самого короткого по суммарной продолжительности альбома в самый длинный.
5. Удаление одним запросом альбомов, не содержащих композиций.

</details>

## 8. Database Design Project — Digital Cookbook

**Темы:** ER-моделирование, древовидные структуры, CTE, оконные функции, транзакции и изменение данных.

### Предметная область

Проект представляет базу данных электронной поваренной книги, которая включает:

- древовидную структуру категорий блюд;
- ингредиенты и их альтернативные названия;
- блюда и входящие в них ингредиенты;
- изображения блюд и ингредиентов;
- инструкции и время приготовления;
- используемую кухонную технику;
- стандартные и нестандартные единицы измерения;
- коэффициенты преобразования единиц измерения.

Количество ингредиентов в блюде указывается в стандартных единицах измерения. Одно блюдо может относиться к нескольким категориям.

### Структура папки

`2.Database Design Dishes: ER-Modeling, DDL Schema Implementation`

| Файл | Содержание |
|---|---|
| `ER-Diagram` | ER-диаграмма электронной поваренной книги |
| `2.SQL script created according to the ER diagram` | DDL-скрипт создания структуры базы данных |
| `3.CTEs and Window Functions...` | Запросы с CTE, оконными функциями, подзапросами, транзакциями и изменением данных |

<details>
<summary><strong>Реализованные запросы</strong></summary>

0. Найти наиболее широко используемый ингредиент или несколько ингредиентов при равенстве результатов.
1. Найти самое тяжёлое блюдо по суммарной массе ингредиентов.
2. Вывести блюда, состоящие только из муки, яиц и сахара.
3. Удалить подсолнечное масло из блюд, в которых одновременно используется оливковое масло.
4. Заменить единицу измерения `чайная ложка` на `грамм` и пересчитать количество по правилу: одна чайная ложка равна восьми граммам.
5. Вывести без повторений название блюда, название ингредиента, количество использований ингредиента во всех рецептах и число ингредиентов в конкретном блюде.
6. Вывести категории, блюда и ингредиенты в алфавитном порядке, пронумеровав ингредиенты отдельно внутри каждого блюда.

</details>

## Что демонстрирует проект

- уверенное владение базовым и продвинутым SQL;
- умение проектировать реляционные базы данных;
- понимание нормальных форм и функциональных зависимостей;
- работу с иерархическими и многотабличными структурами;
- применение подзапросов, CTE и оконных функций;
- изменение данных с сохранением ссылочной целостности;
- анализ текстовых данных с помощью регулярных выражений;
- написание читаемых и структурированных SQL-скриптов.

## Запуск SQL-скриптов

1. Установите PostgreSQL и любой удобный клиент, например `psql`, pgAdmin или DBeaver.
2. Создайте отдельную базу данных для нужного проекта.
3. Сначала выполните DDL-скрипты, создающие таблицы и связи.
4. При необходимости загрузите тестовые данные.
5. Выполняйте аналитические запросы по отдельности в порядке, указанном в файлах.

> Некоторые задания используют учебную схему `employees` и предполагают наличие соответствующих таблиц и тестовых данных.
